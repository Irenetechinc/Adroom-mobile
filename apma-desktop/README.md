# APMA Desktop Dashboard

> Autonomous Political Marketing Agent — Client Desktop App

## What this is
A standalone Electron application (Windows / macOS) that gives political marketing clients a **read-only view** of their APMA campaign. No internal workings or source code are exposed — clients see only:

- **Narrative Score** — current sentiment trend vs. target
- **Sentiment Trend Chart** — 30-day rolling line chart
- **Actions (24h)** — posts, comments, blog articles, group activity
- **Narrative Themes** — top political clusters by volume + sentiment
- **Recommendations** — AI-generated suggestions, with a veto button
- **Blog Network** — list of created sites and their status

---

## Prerequisites
- Node.js 18+
- npm or pnpm

---

## Development (no installer)

```bash
cd apma-desktop
npm install
npm run dev
```

This starts Vite on port 3456 and opens Electron pointed at it.

---

## Building an installer

### Windows `.exe` installer
```bash
cd apma-desktop
npm install
npm run dist:win
```
Output: `apma-desktop/release/APMA Dashboard Setup.exe`

### macOS `.dmg`
```bash
cd apma-desktop
npm install
npm run dist:mac
```
Output: `apma-desktop/release/APMA Dashboard.dmg`

### Both platforms at once (on Linux/macOS CI)
```bash
npm run dist:all
```

## First-run setup

When the app opens:
1. Enter the **Backend URL** (e.g. `https://backend.adroomai.com`)
2. Enter the **APMA API Key** provided by the administrator
3. Click **Connect** — the key is verified against the backend and stored securely in the OS keychain

---

## Authentication
- API keys are 64-character hex strings generated by the admin panel
- Stored securely via `electron-store` (OS-level encrypted storage on macOS/Win)
- Never transmitted to any third party

---

## API Key Management (Admin)

Via the admin panel at `https://your-backend.com/admin`:
1. Go to **APMA → Clients**
2. Click **Create Client** or select an existing one
3. The API key is shown **once** after creation
4. Use **Rotate Key** to issue a new key if compromised

---

## APMA Client API Endpoints (behind the key)

All routes under `POST/GET /api/apma/client/*` require `x-apma-key` header.

| Method | Path | Description |
|--------|------|-------------|
| GET | `/dashboard` | Full dashboard snapshot |
| GET | `/campaigns` | Campaign list |
| GET | `/sentiment-trend?days=30` | Historical sentiment |
| GET | `/recommendations` | Latest AI recommendations |
| POST | `/veto/:recId` | Block a recommendation |
| GET | `/actions` | Recent actions log |
| GET | `/blogs` | Blog network |

## Adding app icons

Place the following files in `apma-desktop/assets/`:
- `icon.ico` — Windows (256x256)
- `icon.icns` — macOS
- `icon.png` — Linux / fallback (512x512)

---

## Environment separation
This desktop app connects **only** to the `POST /api/apma/client/*` namespace.
The AdRoom mobile app has zero awareness of APMA routes. They are completely isolated.
