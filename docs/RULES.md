# Project Development Rules and Standards

This document outlines the strict rules and guidelines that must be followed for all development in this project.

## 1. Data Integrity and Real-time Requirements
*   **No Dummy Data**: The use of dummy or placeholder data is strictly prohibited.
*   **No Hardcoded Data**: Values should not be hardcoded in the source code. Use configuration files, environment variables, or dynamic data fetching.
*   **Real-time Implementation**: All features must be designed for real-time operation using real-time data sources.

## 2. Testing and Quality Assurance
*   **Mandatory Testing**: No task is considered complete until it has been fully tested.
*   **Error-Free Execution**: All code must be verified to be error-free before being executed or deployed.

## 3. Coding Standards and Security
*   **Standard Compliance**: Code must follow established coding standards and conventions for the respective languages and frameworks.
*   **Security First**: Implementation must be security-tight. Vulnerabilities must be proactively addressed.
*   **Best Practices**: Follow industry best practices for maintainability, scalability, and performance.
